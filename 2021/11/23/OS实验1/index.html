​ 进行编译，执行命令sudo make -j8

​ 出现下面的错误：

​ 说明缺少上面的几个东西，apt install libelf-dev安装即可，重新执行make -j8,又出错了，提示是ownership出错，很快就结束了，应该是权限没有及时给，再次执行make -j8,因为上次执行给了权限，这次可以正常执行了，但是中途还是需要获取权限，选择允许。

​ make -j8是指使用8个线程并行编译，这样编译速度要比普通的make命令快很多，考虑不同处理器性能不同，本次编译快则几十分钟，慢的话可能需要几个小时。

​ 叒叒出错了，这次出错情况：

然后试了一下make modules_install，出错：

重新make -j8，出现Kernel:arch/x86/boot/bzImage is ready (#1),这实际上是说明编译已经成功了，并且内核镜像在源码目录下的arch/x86/boot/bzImage，但是后面的make modules_install并不能成功。。。

建议指向上面几条sudo前，先更新当前虚拟机上的软件包，执行命令apt update，apt upgrade这两个命令，如果有很多的软件包没有更新的话，第二个命令会执行较长的时间，需要耐心等待，由于之前因为更新时间太长，我中途结束了更新，虽然后面的apt install命令都可以成功下载到，但make过程中可能出现工具不全。

重装虚拟机，这次先apt update apt upgrade,耐心等待结束，重复上面几个步骤，make -j8一直没报错，过了40分钟，内存不足？？？，70G的硬盘空间，分给文件根目录的只有20多GB，还有好多给了data，但data的空间用不到！！！

尝试使用Gparted调整分区，太菜了，没有成功，还把分区搞乱了。。。

重装虚拟机，尝试自定义分区，系统安装失败，再次启动卡死。。

重装虚拟机，尝试自定义分区，系统安装成功，再次启动卡死。。

重装虚拟机，被迫采用系统默认分区，硬盘空间设成150GB，系统安装成功，再次启动卡死。。

重装虚拟机，被迫采用系统默认分区，硬盘空间设成120GB，系统安装成功，启动成功，先执行apt update apt upgrade,在设置里允许安装任何来源的程序，重复上面几条指令，make -j8成功执行，半夜编译，早上检查make成功，但提示libelf-dev没有安装！！！，虽然后面的make modules_install成功了，但是担心前面的warning会造成后面出错，重新make -j8,这次保证所有用到的，都安装了！！耐心等待。。
